# Incident: 2026-04-08 17-39-26

## Summary      

Between approximately 17:00:00 and 18:00:00 (UTC), users of the JWT Pizza application experienced failures when attempting to place orders. The application returned 500 errors indicating that orders could not be fulfilled by the external Pizza Factory Service.        

The incident was caused by a chaos engineering event that intentionally overloaded or disabled the external factory service. The application itself remained operational but was unable to complete order fulfillment due to the dependency failure.        

The issue was detected via increased latency metrics and error logs showing failed factory requests. After a replication of the problem, the incident was resolved by following the provided chaos report URL and terminating the chaos event.       

This was a moderate severity incident since a core functionality was halted for users during the testing period.         


## Detection        
This incident was detected when monitoring metrics alerted to a spike in failed pizza purchases and application latency. Additionally, application logs showed repeated failures in requests to the Pizza Factory API, returning non-200 status codes.      

The on-call engineer (myself) observed:     
- Increased failure rate in pizzaPurchase(...)
- Error logs indicated "Failed to fulfill order at factory"

Time to detection could be improved by implementing:        
- An alert on failure rate percentage
- An alert on non-200 responses from the factory API

## Impact       
For approximately 30 minutes between 17:30:00 and 18:00:00 UTC-6 on 04/08/2026, users attempting to place orders experienced failures.
- Affected functionality: order creation (POST /api/order)
- User impact: users received 500 errors and could not complete purchases
- Estimated impact: potentially all active users during the chaos window

No support tickets were submitted but logs indicate failures during the incident period.

## Root cause
The root cause of the incident was a chaos engineering event that caused the external Pizza Factory service to become unavailable or reject requests, likely due to simulated overload or exhaustion.       

## Resolution
The incident was resolved by following the provided URL returned in failed factory responses and terminating the chaos event.

Once the chaos event ended:
- Factory API returned to normal operations
- Order requests began succeeding again
- Error rates returned to baseline      

Time to mitigation could be improved by:        

- Faster alerting
- Clearer dashboards showing dependency failures
